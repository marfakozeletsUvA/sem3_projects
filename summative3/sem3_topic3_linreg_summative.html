Multicollinearity (Correlation Matrix)

Why? Highly correlated predictors (multicollinearity) can make linear regression coefficients unstable and hard to interpret. We check for pairs with |correlation| > 0.85.

# check for highly correlated pairs (threshold = 0.85)
print("Checking for multicollinearity -> correlation > 0.85)")
found_high = False

for i in range(len(numeric_cols)):
    for j in range(i + 1, len(numeric_cols)):
        corr = corr_matrix.iloc[i, j]
        if abs(corr) > 0.85:
            print(f"  {numeric_cols[i]} & {numeric_cols[j]}: {corr:.3f}")
            found_high = True

if not found_high:
    print("No highly correlated pairs found, so no features need removal")

GENERALISATION ERROR = BOOTSTRAP CONFIDENCE INTERVALS

One test set gives one error estimate. Bootstrap gives me a range (confidence interval) showing how stable my error is

In[19]:
# bootstrap for confidence interval
np.random.seed(10)
n_bootstrap = 1000
bootstrap_rmse = []

for i in range(n_bootstrap):
    # sample with replacement
    idx = np.random.choice(len(y_test), size=len(y_test), replace=True)
    y_sample = y_test.iloc[idx]
    pred_sample = test_pred[idx]
    
    # calculate RMSE
    rmse = np.sqrt(mean_squared_error(y_sample, pred_sample))
    bootstrap_rmse.append(rmse)

# 95% confidence interval
ci_lower = np.percentile(bootstrap_rmse, 2.5)
ci_upper = np.percentile(bootstrap_rmse, 97.5)
mean_rmse = np.mean(bootstrap_rmse)
